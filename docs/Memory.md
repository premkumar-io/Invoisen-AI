# Engineering Rules

Always: - Write production-ready code. - Follow TypeScript strict
mode. - Keep code modular. - Avoid duplication. - Reuse components. -
Write reusable services. - Document changes. - Keep APIs versioned. -
Preserve backward compatibility when possible.

Never: - Commit secrets. - Break documentation. - Leave TODOs in
production. - Skip validation.
